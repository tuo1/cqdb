Whitepaper

CloudQuery Integrated Cloud Platform for Database Operation and Management

Abstract​

As the pace of informatization construction and application accelerates, data volumes are growing rapidly, the number and types of databases continue to increase, and database environments are becoming increasingly complex. Databases are at the core of data storage and processing, and massive amounts of data are constantly flowing through database systems. If there are security vulnerabilities within, it will directly impact the normal operation of core business and may even jeopardize social stability.

Based on statistics from recent years on database security incidents, the biggest threat to database security comes from internal sources. Malicious insiders and human errors pose the greatest threat to database security, rather than external intruders. Internally, controlling and monitoring database access and operations are crucial for data security to prevent data leakage, data corruption, and database unavailability resulting from internal factors, which are often overlooked.

Innovatively, CloudQuery proposes a solution for "managing database uniformly and safely,using batabase efficiently" ,addressing various aspects of efficient and secure data usage. This ensures data security, prevents leakage of critical data and personal information, and provides assurance for promoting high-quality development in science and technology informatization efforts.

1. Status of Data Security​

CloudQuery has continuously conducted research and identified the following risks in the current enterprise database management model:

Account Security Risks:

  • Unauthorized database accounts may exist.
  • Database accounts of employees who have left the company, resigned, etc., might not be promptly removed.
  • Long-term unused database accounts might be present.
  • Weak passwords or infrequent password changes for database administrators and accounts with sensitive permissions.
  • Database accounts are shared by multiple operators, making it difficult to trace specific actions to individuals.
  • Database account passwords are stored in plain text.

Permission Control Risks:

  • Users or departments may have database permissions beyond their job responsibilities, resulting in a risk of unauthorized use.

Data Leakage Risks:

  • Lack of permission control for data querying and exporting, allowing copying and exporting of data for those with database permissions.
  • Insufficient measures to prevent data leakage through screen capture or photography.
  • Sensitive data such as personal information, business records, and technical information not appropriately masked and displayed in plain text.

Risk of Operational Staff Tampering and Database Overuse:

Operational staff may use database accounts for management with permissions exceeding what is necessary for actual operational management.Without appropriate access behavior control, motivated by personal gain, operational staff could directly manipulate transactions and steal sensitive data.

Insufficient Audit Trails and Difficulty in Tracing:

Monitoring data access throughout the organization is a critical means of tracing and gathering evidence.Inadequate alerting for abnormal data operations and failure to collect detailed audit information on database activities can make tracing difficult, resulting in unclear accountability and potential for recurring security breaches.

Risk of Accidental Data Deletion:

Core business databases are vital for stable business operations, with wide-ranging impacts.Without secure control mechanisms for data deletion or modification operations on core business databases, accidental or malicious operations could lead to irreparable consequences.

2. Approach to Problem Resolution​

Establish a unified database security management platform. This includes a unified database access entry, unified database user identity authentication, unified database permission control, unified database monitoring and auditing, as well as capabilities for data masking, SQL review and deployment, data backup and rollback, and alerting and interception of high-risk operations. This system aims to prevent unauthorized data access, data leaks, accidental data deletion, etc., ensuring the smooth and efficient operation of the database. The primary objectives are as follows:

Unified Database Access Entry:

By providing a unified platform for database development and management, we aim to centralize the database access entry and management entry, while implementing unified and secure control over individual data access behaviors.

Fine-grained permission control：

This involves precise control over database access behaviors, such as controlling the operation object, operation type, operation time, and terminal window before access, and managing high-risk operations and sensitive data protection during access. It also includes controlling resultset operations and export functions after access. This helps prevent the misuse of high-privileged accounts, limit unauthorized operations, and mitigate high-risk actions.

Prevention of Data Leakage:

Restrict users to access data within their own permissions, following the principle of "separation of querying and exporting" where querying and exporting permissions are granted separately. This prevents internal data leaks, database dragging, and table dragging. Additionally, features like data copy switch and data watermarking are included to prevent sensitive data from being copied or leaked through screenshots or photographs.

Protection of Sensitive Data:

Desensitize sensitive data such as personal information, business records, and technical information before display, adding an extra layer of security. This protects privacy and confidential data without affecting normal data access.

Prevention of Accidental Data Deletion:

Implementing closed-loop management for data change processes. Prior to data changes, providing risk alerts, SQL reviews, manual reviews, data backup, and other operations. If users delete data accidentally,the platform provides data recovery function. Comprehensive control is exercised over data change before, during, and after, to prevent accidental data deletion.

Enhancement of Development Efficiency:

  • Unified database operating interface for centralized management of various database types. No need to switch between multiple database management tools frequently, enabling unified operations across different types of databases without the need to purchase multiple database client licenses, thus saving usage and learning costs.
  • Conduct pre-deployment review of SQL to prevent potentially risky SQL from going live.
  • Different departments can engage in online process handling, eliminating the need for users to engage in inefficient offline communication.

Security Audit:

Conduct a comprehensive and precise analysis and recording of data operations, analyzing data operation behaviors at three levels: overview, user, and database object.

  • Display the volume of operations in a graphical interface, making database operation behaviors clear at a glance.
  • Aggregate user behaviors and permissions at the user level, tracking specific user behaviors in real-time.
  • Aggregate the specific actions performed on individual database objects at the database object level, and track the actions on database objects in real-time.

This enables traceable and queryable data operation trails.

5. Product Features​

5.1 Unified Access Entry​

Unified Database Management. CloudQuery Provides a unified platform for database development and operations, achieves a unified database access entry and management entry. Enterprises can achieve unified and secure control over data access behaviors.This includes: unified database access entry, unified database user identity authentication, unified database permission control, and unified database monitoring and auditing.

CloudQuery can centrally manage enterprise databases within the platform by means of account retrieval, restricting access IPs, and limit
ing access tools, thereby consolidating other database access entries.The platform serves as a unified access entry for databases, providing users with a convenient and fast data querying method based on B/S architecture SQL interaction. Users can access all databases through a single web page. CloudQuery can enhance data operation efficiency while shielding the differences between various databases, saving users time and learning costs associated with switching between different database management tools.

5.2 Data Operation​

As a cloud-oriented database security and control platform, the cloud database client is the core module of the platform. Its significance is akin to the kernel of an operating system, upon which all other platform functionalities are built and extended.It has the following core functionalities:

  • Advanced SQL Editor: An advanced SQL editor that spans multiple types of database. The editor supports SQL 2011 standards and database-specific syntax. It allows for quick switching between different databases, eliminating the need for repetitive switching between various clients. It features code highlighting, auto-completion, SQL formatting, execution plans, favorites, shortcuts, graphical interface, and other rapid coding assistive tools.

  • Database Object Management: The platform supports the creation and editing of over ten types of database objects, including databases, tables, functions, stored procedures, DBLINK, packages, package bodies, sequences, triggers, and more.

  • Data Import and Export: The platform supports data import from files such as EXCEL, CSV, TXT, etc., with customizable delimiters. It also supports the export of database structure, table structure, and table data.

  • Graphical Visualization: Supports a graphical interface for creating tables, indexes, and triggers. It also enables changes to table structures and CRUD operations on table data through a graphical interface. The operation is consistent with traditional clients, making it easy to use.

  • Database Terminal Management: Supports database terminal connections. When the editor cannot meet the requirements, database-related operations can be performed using the terminal connection method, covering all database commands.

  • Resultset Management: Resultset data is displayed in a paginated form, and you can set the number of rows per page to improve query efficiency and reduce server load. The platform supports resultset viewing, editing, copying, and downloading. It also supports single-row data viewing mode, binary field display, and execution log display, showing execution time, results, executed SQL, errors, and other information.

  • Data Dictionary Search: It supports searching the database elements through a data dictionary. For example, searching by "data element name," "data element name keywords," "data element name list," etc., to accurately, quickly, and in batches filter out data elements. It should support filtering for "hundreds of thousands" of database elements and be able to filter multiple data elements in one batch.

5.3 Privilege Escalation Process​

Users can apply for permissions to administrators through the platform's privilege escalation process, enabling privilege escalation. The privilege escalation process meets the internal system approval process needs of the customer, and supports customizable approval processes. It allows for adjustment of approval nodes, enables setting multiple approvers for a single node for user selection, and supports adjustments based on approver types. Users can choose the duration of the permissions during the application process, and the permissions are automatically revoked upon expiration. Specific types of permission requests include:

  • Access Request: Users can request database access through an access request. The access request supports resource search, allowing the addition of different types of resources and selection of varying permission levels (database operation sets) for a single access request. Users can also choose the effective time for the permission level. After initiating the request, users can withdraw it. Upon completion of an access request, users can reapply.

  • Operation Privilege Escalation: When users need to access specific database objects (tables, views, functions, etc.), they can directly initiate an operation privilege escalation on the data operation SDT tree object for unauthorized operations (DML, DQL, DML). A single operation privilege escalation supports adding different types of database objects and selecting the effective time. Users can initiate, withdraw, reapply, or make new applications after initiating an operation privilege escalation.

  • High-Risk Operation Permission Request: When users need high-risk operation permissions, they can apply through a high-risk operation application. Users can selectively initiate a high-risk operation application based on the high-risk levels defined by the administrator when encountering high-risk operations. They can choose the effective time for the high-risk operation. After initiating the high-risk operation application, users can withdraw and reapply it.

  • Tool-use Permission Request: Users can request terminal window access, synchronous data dictionary access, resultset operations, and resultset export through a tool permission request. Users are prompted to apply when they lack permissions. After initiating the request, users can withdraw it and reapply.

  • Data Change Request: When users need to make data changes, they can apply through data correction or change publication workflows. The data-change workflow supports bulk script entry or upload, SQL checking, and rollback statement entry. The rollback statement is used for rollback operations in case of script execution failure. Users can choose between manual and automatic script execution modes.

  • When users need to view sensitive data in plaintext, they can apply through the database data desensitization privilege escalation process. After a successful privilege escalation, users can view sensitive data in plaintext. During the application, users need to select the fields they wish to view in plaintext.

5.4 Audit analysis​

Traditional database audit products are generally deployed on the side, analyzing mirrored switch traffic. However, they face issues such as duplicate analysis, incorrect analysis, and long statement splitting analysis for private database protocols.

This leads to millions of records of behavior every day, overwhelming operations personnel and making it impossible to extract effective operational records from the massive audit records. They are unable to rely on audit results for decision-making, rendering database audit practically useless.

CloudQuery effectively addresses the shortcomings of traditional database audit products. It can obtain executed SQL statements based on the platform's underlying interface, resulting in audit results that are 100% accurate.

  • Audit Overview. Leveraging the built-in streaming analysis engine, the CloudQuery platform can analyze and determine user behavior, intercept high-risk user operations, issue alerts, and create a user profile of data operations. For database actions, CloudQuery conducts real-time monitoring during operations and comprehensive post-operation auditing. The audit content covers system management and security behavior, retaining logs and records. It can record information such as the SQL executor's account, IP, statement, execution results, duration, affected rows, and more. All actions taken by a user upon logging into the platform are recorded, presenting metrics like total executions, errors, active users, average SQL duration, and more.Audit information can be analyzed from various dimensions, allowing a detailed view of system personnel's operational behavior from different angles. By dynamically monitoring user actions and presenting detailed behavioral records and security trends through a user-friendly interface, it reduces the workload of operations personnel in auditing and achieves precise behavioral auditing to ensure enterprise data security.

  • Object Audit. This is an independent audit function focusing on database objects. Users can specify the type of problem and database objects for auditing, enabling rapid identification of when and by whom specific operations were performed on database objects. It presents audit information from the perspective of database objects.

  • User Audit. This is an independent audit function focusing on user objects. It monitors users' actions and access behaviors, including login and logout activities, operations and access to the database, permission changes, and more.

5.5 Data Protection​

Dynamic Data Masking. Based on important data security laws and regulations, policies such as requirements for sensitive system data, enterprises need to classify and set desensitization strategies for internal basic data, and build hierarchical dynamic desensitization capabilities to achieve dynamic desensitization of sensitive data, adding an extra layer of security to sensitive data.

Enterprises do not need to alter the data in the production database but can differentially desensitize the data returned by the database based on user and data levels. CloudQuery's dynamic desensitization feature supports desensitization algorithms such as random mapping, masking, randomization within a range, random fluctuation, truncation, and data simulation. The CloudQuery dynamic desensitization module has the following core functions:

  • Data Desensitization. Based on predefined desensitization strategies, the platform automatically desensitizes the presented resultsets. This rule applies to all data and all users without plaintext data query permissions. In addition to some common built-in desensitization strategies, users can define their own desensitization rules.
  • Specific Field Desensitization. Administrators can set field desensitization on specific database objects, specifying fields for desensitization. Once enabled, this affects all users on the platform. If desensitization shouldn't apply to certain users, administrators can configure policies to specify users with certain attributes to view data without desensitization or to exempt desensitization during specific time periods.
  • Data Export Desensitization. During data export, the data is automatically desensitized based on built-in rules and configured desensitization strategies.
  • Automatic Discovery of Sensitive Data. Database administrators can create scanning tasks and, based on scanning rules, discover and locate sensitive data from a large amount of data resources quickly. The discovered sensitive data is then presented to the administrator, who decides whether to desensitize the data.

Row Filtering. Rule settings are based on different parameters for various users, achieving varied results for viewing the same data based on each user's settings.

5.6 Data Modification Management​

Management of Data Modification Processes, achieving closed-loop control over data alterations. CloudQuery can comprehensively control data changes before, during, and after the process to avoid database fluctuations. It has the following specific capabilities:

  • Risk Alerting Capability: When initiating a data change request and during the approval process, the platform can provide risk alerts for SQL scripts, including alerts for high-risk SQL statements, affected row count alerts, syntax analysis and error SQL diagnosis, and execution plan alerts.
  • Platform-Based Execution Capability: For SQL scripts that have been approved, the platform can automatically execute them at scheduled times or limit manual execution within a specified time range. It also tracks the execution status and audits the execution results.
  • Data Backup Capability: Before executing data change scripts, the platform can automatically back up the modified data. The backup location can be set by the administrator.
  • Alerting Capability: The platform can generate alerts for high-risk statements, abnormal user behavior, and high-impact statements. It can analyze from various dimensions such as database, user, executed SQL, and generate statistical charts.
  • Configuration Capability: Administrators can configure audit rules and approval processes, including configuring multi-person review processes, defining rules for high-risk SQL, and setting thresholds for affected row counts.

5.7 Fine-grained Permission Control​

CloudQuery utilizes self-developed permission middleware, implementing role and permission control based on the RBAC and ABAC models. Different operational permissions are assigned based on individuals' positions, responsibilities, and departments, achieving database permission separation and fine-grained permission control.

  1. It supports fine-grained permission control over users, IPs, tables, columns, rows, operations, statements, time, and more.
  2. Various control methods are supported, including allow, alert, block, data masking, etc.
  3. It supports restricting returned fields, limiting execution times, restricting execution duration, and restricting the target of operations, fundamentally addressing the issue of misuse of data operation permissions by personnel.
  • Connection Management. A connection is a channel created when accessing the database. The platform uses Schema as the smallest container unit for databases, isolating resources between production and non-production environments, ensuring the accuracy and practicality of managed resources. It supports viewing the current moment's connection occupancy, connection usage, and configuring relevant properties of the connection, such as maximum connection count, maximum wait time, idle connection timeout, etc.
  • Manual Authorization. Satisfies user authorization for small-scale scenarios, such as granting specific database objects to users. User authorization based on the database object. Similar data sources can have customizable permission levels (i.e., sets of operation types: DDL, DML, DQL), and these permission levels can be granted to users on specific database objects, giving users permission to operate on that database object. Supports fine-grained authorization on specific database table, view, and function objects.
  • Automatic Authorization. Satisfies authorization in most scenarios, integrating batch authorization and automatic authorization. Different types and levels of permissions are granted to users that meet the predefined policy conditions. When a user loses certain attributes configured in some policies, they automatically lose those permissions. Supports packaging permissions of different types of data sources into a permission set and automatically triggering authorization to users based on policy conditions. Permissions are revoked when conditions are not met, catering to most authorization scenarios.
  • Hierarchical Authorization Management. This involves assigning administrators to resources. Becoming an administrator for a resource automatically grants control over that resource, allowing for authorization management, security settings, and data protection within the management scope, sharing the responsibilities of a connection administrator.

5.9 Data Export​

CloudQuery has an enterprise-level data export capability that strictly adheres to data protection regulations. This capability includes fine-grained permission control, such as query-export separation and field-level export object control, as well as rigorous behavior auditing with export records and data automatically archived for audit personnel to perform periodic inspections. It also features the ability to export millions of rows of data under the control of a multitask engine. Specifically, it has the following capabilities:

  • Support for Multiple Database Types: Supports mainstream databases such as Oracle, HBase, MySQL, GaussDB, etc.
  • Permission Control: Separation of query and export permissions, with control over the exported data objects (down to the field level), export time (down to the minute), and the number of exported rows based on the exporter's permissions.
  • Export Behavior Audit: Records initiator of the export, client IP, SQL statement, execution time, export time, number of affected rows, and other information related to the export behavior. Audit personnel can search and view export records and exported data at any time.
  • Large-scale Data Export: Capable of stable and fast export even with data sizes in the order of billions of rows.
  • Support for Multiple File Formats: Supports export to file formats such as PDF, Excel, TXT, CSV, SQL, and more.
  • Support for Background Tasks: Export tasks are executed in the background, not affecting the frontend operation. Even if you exit the platform, it does not interfere with the execution of export tasks.
  • Support for Frontend and Backend Export Modes: Can export data that is already loaded in the frontend, without the need to repeat requests to the server.

In addition, each level of departments and teams within the organizational structure has its own independent folder. 6. Key Benefits​

6.1 Simplified Database Account Management​

When using CloudQuery, users only need to create a single CloudQuery account, eliminating the need to create an account in each database and grant permissions. This design significantly simplifies database account management, as users can focus on managing the CloudQuery account, greatly reducing the workload for administrators. The CloudQuery account system is straightforward and user-friendly, supporting multiple login methods (such as account/password, phone number, third-party authorization, etc.) that users can choose based on their preferences. This provides a convenient and secure user authentication experience. The CloudQuery account system also supports various identity and permission controls, allowing administrators to manage and authorize user access to different data resources securely.

CloudQuery offers comprehensive account management functionalities. Users can maintain their accounts, reset passwords, view login logs, set security policies, and perform other actions according to their needs, ensuring the security and compliance of their accounts. In summary, CloudQuery's standalone account system effectively addresses the challenges of database account management, simplifying the management and operational processes while enhancing efficiency and security.

6.2 Prioritize Data Security

The data protection feature of CloudQuery primarily relies on pre-parsing SQL syntax to analyze and intercept database operations beforehand, resulting in an innovative data protection approach. The specific implementation is as follows:

  • SQL Syntax Parsing: Through SQL syntax parsing, CloudQuery accurately recognizes keywords, types, table names, column names, and other information within SQL statements. Additionally, it can analyze the syntax to determine the type, security, and compliance of the SQL.
  • Interception of SQL Statements: Based on the results of SQL syntax parsing, CloudQuery can match relevant data protection rules, such as identifying risks like table locking, multi-table queries, and password exposure. If a security risk is detected in the SQL statement, CloudQuery intercepts the SQL statement to prevent data damage or leakage.
  • Access Control: CloudQuery supports access control for database objects by providing database access authorization and limiting user data access permissions. It also offers real-time monitoring and alerts, detecting malicious access or abnormal operations to precisely protect the database.

CloudQuery's data protection approach, employing SQL syntax parsing and interception control, realizes a novel data protection and defense mechanism, ensuring data security and compliance. With precise matching and access control of SQL statements, it prevents events such as data leaks and accidental operations, enhancing data security and reliability. Moreover, CloudQuery's data protection approach supports various data encryption modes, such as row filtering and column-level encryption, to encrypt and safeguard sensitive data within the database, ensuring the security of data privacy and confidentiality.

6.3 Leading Product Form​

Database operators use SQL to interact with databases, enabling data operations. Therefore, the core technology of database security products lies in accurately parsing SQL and identifying, analyzing, intercepting, and alerting on high-risk and unauthorized behaviors.

Traditional security vendors use a database protocol parsing approach based on network traffic monitoring, which is prone to issues such as inaccurate or lost data. Additionally, this approach, relying on database protocol analysis and cracking, cannot cover new versions of databases and requires high precision in database protocol analysis. It generates a large amount of dirty data when dealing with new versions or databases with private protocols.

In this solution, a self-developed database client approach is employed. It can accurately record and parse SQL executed by users on the platform, achieving 100% accuracy. This greatly distinguishes it from the vast amount of unusable and unreadable data generated by traditional database protocol analysis.

Furthermore, traditional data security management tools can only provide security control capabilities without improving data usage efficiency. They rely on the centralized traffic control model, which is no longer suitable for the increasingly complex multi-cloud database environment. In the wave of the era heading towards a diverse and hybrid database era, we need a unified database management platform that spans multiple clouds, databases, and endpoints.

7. Deployment​

The product form of CloudQuery is software, deployable on cloud servers, physical servers, and virtual machines. It supports single-machine deployment, distributed deployment, HA deployment, and more. It is recommended to deploy in an intranet environment, ensuring data remains within the intranet and enabling complete control over security and data ownership within the environment. Deployment does not require changes to the database or server configurations, allowing for one-click deployment and seamless access within seconds.
